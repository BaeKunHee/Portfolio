04_statistical_analysis_examples — 구조 검증을 위한 기초 분석 예시
1. 목적 (Goal)

본 단계는 모델 성능 향상이 목적이 아니다.
앞선 단계(수집 → 정제 → 데이터셋 구축)가 올바르게 수행되었는지를 기초 통계 분석을 통해 확인하는 단계이다.

특히, 분석의 초점은 다음에 있다:

(TM, STN) 단위가 유지되었는지 확인

변수 선택이 왜곡 없이 평균 구조를 반영하는지 검증

데이터 구조 변경 시 결과가 달라질 수 있음을 확인

즉, 분석은 “목적”이 아니라 데이터 구조의 타당성 검증 수단이다.

---

2. 분석 내용
STN(관측소)별 평균 기온(TA) 및 평균 풍속(WS) 비교

사용 데이터:

datasets/hourly_base.parquet


분석 과정:

df = pd.read_parquet("datasets/hourly_base.parquet")

stn_mean = (
    df.groupby("STN")[["TA", "WS", "HM"]]
      .mean()
      .reset_index()
)


관측소(STN) 단위로 그룹화

평균 기온(TA), 평균 풍속(WS), 평균 습도(HM) 계산

집계 단위는 여전히 STN 기준으로 명확히 정의됨

---

3. 시각화
plt.figure(figsize=(8, 4))
sns.scatterplot(data=stn_mean, x="TA", y="WS")
plt.xlabel("Mean Temperature (TA)")
plt.ylabel("Mean Wind Speed (WS)")
plt.title("STN-level Mean Comparison")
plt.tight_layout()
plt.show()


시각화 목적:

관측소별 평균 기온과 평균 풍속 간 관계 탐색

이상하게 튀는 관측소가 존재하는지 점검

평균 구조가 논리적으로 일관적인지 확인

---

4. 왜 이 분석이 중요한가

이 단계는 다음을 점검하기 위한 것이다:

분석 단위 일관성

(TM, STN) 구조가 유지된 상태에서 집계가 수행되었는지 확인

결측 처리 정책의 영향

placeholder → NaN 변환이 평균 계산에 어떻게 반영되는지 확인

변수 선택의 적절성

CORE_VARS만으로도 기초 통계 구조가 합리적인지 검증

데이터 구조 변경 시 결과 변화 가능성 인식

예: cleaning 정책이나 변수 subset이 달라지면 평균값도 달라질 수 있음

본 분석은 단순 탐색적 비교이며,
통계적 가정은 이후 단계에서 정의된다.

---

6. 데이터 품질 관점 해석

본 단계는 데이터 품질을 다음 관점에서 점검한다:

평균 계산이 정상적으로 수행되는가?

STN 단위 집계 시 중복 문제가 없는가?

특정 관측소가 비정상적으로 분포에서 벗어나지는 않는가?

cleaning 정책이 평균 구조를 왜곡하지는 않는가?

이는 “결과 해석 이전에 구조를 점검하는 과정”이다.

---

7. 설계 철학

본 프로젝트에서 분석은 목적이 아니라 다음을 확인하는 수단이다:

데이터 구조가 바뀌면 결과도 달라진다.

따라서 본 단계는,
앞선 설계 결정(정제 정책, 변수 선택, 단위 고정)이
실제 통계 요약값에 어떤 영향을 미치는지 확인하는 검증 단계이다.
